Candidates: HDFS - Hadoop (Java) CloudStore Lustre - special kernel- http://wiki.lustre.org/index.php/Main_Page MogileFS - NOT POSIX Compliant -- you don't run regular Unix applications or databases against MogileFS. It's meant for archiving write-once files and doing only sequential reads GlusterFS - NAS file system - http://www.gluster.org/ - Used by RedHat for a new Appliance, should work!! MooseFS - POSIX compliant - http://www.moosefs.org/

Use S3: http://code.google.com/p/s3backer/ - Simple solution, not a file system, good enough if one server writes and the others only read s3fs * http://ubuntuservergui.com/ubuntu-server-guide/mount-s3-ubuntu-server

Dropbox: * http://ubuntuservergui.com/ubuntu-server-guide/install-dropbox-ubuntu-server
